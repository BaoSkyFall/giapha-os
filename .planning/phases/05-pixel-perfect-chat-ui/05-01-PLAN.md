# Plan 05-01: Route Scaffold — Layout & Page

## Objective
Create the Next.js route scaffold for `/dashboard/ai-advisor` using a route group `(ai-advisor)` that bypasses the existing `DashboardLayout`. The layout provides auth guard + Google Fonts + Material Symbols. The page is a server component that pre-fetches the session list.

## Key Architectural Decision
`app/(ai-advisor)/dashboard/ai-advisor/` — route group parenthesis syntax bypasses all parent layouts. This gives the AI Advisor its own full-screen context without the existing header/sidebar.

Create `app/(ai-advisor)/dashboard/ai-advisor/page.tsx` (server component that pre-fetches sessions, then renders the client shell):

```tsx
import { getSupabase, getUser } from "@/utils/supabase/queries";
import AiAdvisorClient from "./AiAdvisorClient";

export default async function AiAdvisorPage() {
  const user = await getUser();
  const supabase = await getSupabase();

  const { data: sessions } = await supabase
    .from("chat_sessions")
    .select("id, title, created_at, updated_at")
    .eq("user_id", user!.id)
    .order("updated_at", { ascending: false });

  return <AiAdvisorClient initialSessions={sessions ?? []} />;
}
```
